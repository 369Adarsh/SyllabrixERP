## Problem

There is no dedicated place in Syllabrix Nerve Center to raise, track, and manage Change Requests (CRs) or Enhancement requests before development begins. Currently the TR creation form has document fields embedded inside it, but there is no standalone CR/ENH lifecycle — no list, no dedicated create form, no document generation, and no formal linkage between a CR/ENH and the TR that executes it.

This means:
- There is no formal intake registry of what changes have been requested and why
- There is no generated document file that travels from Nerve Center to Claude Code
- TRs have no mandatory link to the CR/ENH that authorized them
- The Changes Made file (execution manifest) has nowhere to live in the system

---

## Solution

Build a dedicated **CR / Enhancement** section in the Nerve Center sidebar. This section manages the full lifecycle of a change request — from idea to approved document — before development begins. When a CR/ENH is approved, Nerve Center generates a structured `.md` document file that the user downloads and uploads to Claude Code. After development, Claude generates a Changes Made file. The TR creation form is updated to require both the CR/ENH number and the uploaded Changes Made file before a TR can be created.

## In Scope

### Database
- New `ChangeRequest` model with fields: id, crCode (CR-YYYY-NNN or ENH-YYYY-NNN), type (CR | ENHANCEMENT), title, description, problem, solution, inScope, outOfScope, businessTypeCode, modulesAffected, priority, status (DRAFT | APPROVED | IN_DEVELOPMENT | COMPLETED | REJECTED), createdBy, approvedBy, approvedAt, linkedTRId, createdAt, updatedAt
- New `CRLog` model for audit trail: id, crId, action, performedBy, notes, createdAt

### Backend API (new endpoints under `/api/platform/changes`)
- `GET /` — list all CRs and Enhancements (filterable by type, status)
- `POST /` — create new CR or Enhancement
- `GET /:id` — get single CR/ENH with logs
- `PATCH /:id/approve` — approve a CR/ENH (DRAFT → APPROVED)
- `PATCH /:id/reject` — reject a CR/ENH (DRAFT → REJECTED)
- `GET /:id/document` — generate and download the CR/ENH document as `.md` file
- `GET /stats` — counts by status and type

### Frontend Pages (4 new pages)
- `/platform/changes` — CR/ENH list page (filterable table, stats strip, New CR / New ENH buttons)
- `/platform/changes/new` — Create form (type selector: CR or Enhancement, all document fields)
- `/platform/changes/:id` — Detail page (full document view, approve/reject buttons, Download Document button, linked TR display)
- Sidebar entry: "Change Requests" under a new "Change Control" group or under "Release"

### TR Creation Update
- `TransportNew.jsx` — add CR/ENH Number field (text input, validated against existing CR/ENH records)
- `TransportNew.jsx` — add Changes Made file upload field (accepts `.md` file)
- Backend TR create endpoint — store `crNumber` and `changesMadeFile` (text content) on the TransportRequest
- TR Detail page — display linked CR/ENH number (clickable link to CR/ENH detail) and Changes Made file content

### Document Generation (GET /:id/document)
Backend generates and serves a `.md` file with this structure:
```
# CR-2026-001 / ENH-2026-001
Type | Title | Business Type | Module | Priority | Approved By | Approved At
Problem | Solution | In Scope | Out of Scope
```
File is served as a download attachment with correct filename.

### Prisma Schema
- Add `crNumber String?` and `changesMadeFile String?` to TransportRequest model

## Out of Scope

- No email notifications for CR/ENH approval or rejection (Phase 5)
- No WhatsApp notifications for CR/ENH events (Phase 5)
- No automatic conflict detection between TRs sharing the same changed files (Phase 2)
- No per-tenant CR/ENH raising — this is internal Nerve Center only
- No CR/ENH template library (future enhancement)
- No changes to existing TR Kanban board, promote, or rollback flows
- No changes to any tenant-facing modules

## Notes

- CR code format: `CR-YYYY-NNN` (e.g. CR-2026-001) — separate sequence from TR codes
- ENH code format: `ENH-YYYY-NNN` (e.g. ENH-2026-001) — separate sequence from CR codes
- The Changes Made file is generated by Claude Code after development and uploaded by the user when creating the TR — it is stored as plain text in the database
- This ENH-2026-002 is itself the first CR/ENH that will be tracked in the new system once built
