India has 15 crore+ freelancers and gig workers.
No software has ever been built truly for them —

Syllabrix Freelancer owns this gap.

Every solo earner in India — regardless of profession —
should be able to open Syllabrix and feel it was built
specifically for them.

## THE ID SYSTEM

### Format
```
SYL-FL-[CAT]-[##]-[###]

SYL   → Syllabrix platform
FL    → Freelancer division
CAT   → Category 3-letter code
##    → Category number (01-99)
###   → Individual type number within category (001-999)
```

### Rules
- IDs are permanent. Never renumber or remove.
- New types always append at the end of their category.
- Category codes are locked once assigned.
- Same IDs carry over to mobile app, API, analytics.

## MODULE ARCHITECTURE

### Universal Modules — Every Freelancer Gets These
```
✓ Invoice / Bill (speaks their language)
✓ Expense Tracker (What I Spent)
✓ Client / Customer List
✓ Payment Collection (UPI / Cash / Bank)
✓ Income Dashboard (What I Earned)
✓ WhatsApp Integration (reminders, invoices)
✓ Tax Summary (ITR ready)
```

### Category-Specific Modules — Built Once, Used by Many
```
JOB CARD          → TRD (Trades)
PROJECT TRACKER   → DGT, CRE, CNW, MED, ART
BOOKING CALENDAR  → PHV, EVT, WEL
STUDENT + BATCHES → EDU
BRAND DEAL PIPELINE → CCR
CASE MANAGER      → PRO
TRIP LOG          → GIG
PORTFOLIO TRACKER → INV
CONTENT CALENDAR  → CCR, CNW
TEAM / HELPERS    → TRD, EVT, EDU
AGENCY COMMISSION → ART
```

### Module Reuse Map
```
Booking Calendar is built ONCE and used by:
→ PHV (Photography & Video)
→ EVT (Events & Entertainment)
→ WEL (Wellness & Healing)

Project Tracker is built ONCE and used by:
→ DGT (Digital & Technology)
→ CRE (Creative & Design)
→ CNW (Content & Writing)
→ MED (Media & Journalism)

### The Complete Job Lifecycle
```
ENQUIRY        → Customer calls / WhatsApps about work
SITE VISIT     → Freelancer visits, assesses scope
ESTIMATE       → Labour + materials breakdown, shared via WhatsApp
ADVANCE        → Partial payment collected upfront
JOB STARTS     → Materials purchased, helpers assigned
IN PROGRESS    → Daily material log, helper attendance, partner splits
COMPLETION     → Work done, customer verifies
FINAL BILL     → Auto-generated from job data, advance deducted
PAYMENT        → Balance collected (cash/UPI)
POST JOB       → Warranty starts, referral captured, AMC if applicable
```

#### Dashboard `/freelancer/dashboard`
```
- Hi [Name] + current month

- Completed this month count

#### Jobs List `/freelancer/jobs`
```
- Tabs: ALL | ACTIVE | PENDING PAYMENT | COMPLETED
- Each card: Job ID, Customer, Work type, Location,
  Status badge, Job value, Amount pending, Days running
- Quick actions per card: Call | WhatsApp | View

```
Step 1: Customer (existing or new) + site address
Step 2: Work type, description, dates, advance required
Step 3: Estimate builder

  - Material items (name + qty + unit + rate)
  - Auto-calculated total
  - Advance requested
Actions: Save Draft | Share WhatsApp | Create Job

### Contract Size Tiers
```
SMALL  (₹500 - ₹5,000)   → Fix tap, repair switch, paint 1 room
                            Instant Book — no bidding needed
                            Same day or next day

MEDIUM (₹5,000 - ₹50,000) → Full bathroom plumbing, house painting
                            Quote-based — freelancer sends estimate
                            2-7 day jobs

BIG    (₹50,000+)          → Full house wiring, construction project
                            Tender-based — detailed proposal required
                            Multi-week, milestone payments
                            May need team / partners
```

---

### Milestone Payments (Big Contracts)
```
Example: Full Building Wiring — ₹2,00,000

Milestone 1: Foundation wiring complete  → ₹50,000
Milestone 2: First floor complete        → ₹60,000
Milestone 3: Second floor complete       → ₹60,000
Milestone 4: Final testing & handover    → ₹30,000

Each milestone:
→ Freelancer marks complete
→ Customer verifies
→ Payment releases
→ Next milestone begins
```

Freelancer tab also changes:
- Page heading → "Create Your Freelancer Account"

## OPEN DOOR PRINCIPLES
*Applied to every module during development — never close a door*

```
1. DATA EXTENSIBILITY
   Every DB model includes a `meta` JSON field
   → Add any extra data in future without schema migration

2. FEATURE FLAGS
   Every module has feature toggles
   → Turn on/off any feature without code changes
   → Add new features without breaking existing ones

3. LANGUAGE CONFIG
   All labels, headings, button text from a config file
   → Change any word for any persona without touching components
   → Hindi/English/Regional language support ready

4. CATEGORY EXTENSIBILITY
   Expense categories, payment modes, job statuses — all configurable
   → Add new options from Settings, no hardcoded limits

5. MODULE PLUGGABILITY
   New module = new folder + register in config
   → Core app never needs modification to add a module

6. REPORT PLUGGABILITY
   New report = new file
   → No changes to report engine or core
```
